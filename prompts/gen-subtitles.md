首先向用户询问如下内容以便继续处理：

1.  需要处理哪个音频文件？

2.  以哪个文案文件作为参考？

向用户获取以上必要的信息后，执行以下操作：

1.  激活 `.venv` 虚拟环境

2.  运行 `python utils/extract_audio_subtitles`，并将用户需要处理的音频文件路径作为按位置参数传入

    该脚本会获得每行一个单词的内容（`word`），起止时间（`range`）以及识别的置信度（`conf`）

3.  运行 `python utils/extract_plain_text`，并将文案文件路径作为按位置参数传入

    该脚本会提取输出文案文件中需要用到的纯文本内容，供后续校对使用

4.  将上一步脚本输出的若干单词合并为完整的字幕短句（不包含短句最末尾的标点符号），并给出每个短句的起止时间（`range`），根据文案文件的内容进行校对
    
    如果你合并的短句没有令人困惑的句子则优先采用，否则参考文案文件中的内容进行修正

    对于文案中出现的 `{数学公式}` 占位，如果在语音中包含数学公式片段（可能是比较杂乱的识别结果），请在字幕中同样使用 `{数学公式}` 占位

5.  将处理后的字幕短句以 JSON 格式输出到对应文件中，例如原音频文件叫 `test.mp3`，则输出到同目录下的 `test.json` 中，格式如下：

    ```json
    [
        {
            "text": "字幕短句内容",
            "range": [起始时间, 结束时间],
            "contains_math": 是否包含数学公式占位
        },
        ...
    ]
    ```
